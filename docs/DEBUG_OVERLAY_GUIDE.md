## 🔍 诊断移动问题

### 测试 1: 点击"聚焦 Canvas"按钮

1. 点击调试面板上的 **"聚焦 Canvas"** 按钮
2. Canvas 周围会出现绿色边框
3. 现在**右键点击地面**

**预期结果**:
- 日志区域显示：
  ```
  🖱️ 右键点击 - 移动/攻击：{x: 400, y: 300}
  🎯 玩家移动到 (400, 300)
  📍 移动目标已发送
  ```
- 角色开始移动

---

### 测试 2: 点击"检查玩家"按钮

1. 点击 **"检查玩家"** 按钮
2. 查看日志区域

**正常情况**:
```
📍 玩家数据：{"id":"...","x":100,"y":200,...}
```

**异常情况**:
```
❌ 玩家数据不存在
```
→ 说明未登录或登录失败

---

### 测试 3: 观察 Canvas 焦点

查看调试面板上的 **Canvas 焦点** 状态：

- ✅ **已聚焦** = Canvas 可以接收输入
- ❌ **未聚焦** = Canvas 无法接收输入

**如果显示"未聚焦"**:
1. 点击"聚焦 Canvas"按钮
2. 或左键点击游戏画面
3. 然后再试右键移动

## 📊 日志说明

### 移动相关日志

| 日志 | 含义 |
|------|------|
| `🖱️ 右键点击` | 接收到右键点击 |
| `🎯 玩家移动到` | 开始处理移动 |
| `📍 移动目标已发送` | 移动指令已发送 |
| `🚫 UI 打开中` | UI 遮挡，阻止输入 |
| `❌ 玩家数据不存在` | 玩家未登录 |
| `❌ Canvas 不存在` | Canvas 未初始化 |

## 🛠️ 常见问题

### Q1: 调试面板不显示？

**A**: 刷新页面（Ctrl + Shift + R）

---

### Q2: 点击"聚焦 Canvas"后还是无法移动？

**检查顺序**:
1. Canvas 焦点状态 → 应该显示 ✅
2. 点击地面是否有日志 → 应该有 `🖱️ 右键点击`
3. 玩家数据是否存在 → 点击"检查玩家"

---

### Q3: 有日志但角色不动？

**可能原因**:
1. 网络延迟 → 检查服务端
2. 玩家数据未同步 → 刷新页面
3. 移动被阻止 → 检查是否有 UI 打开
