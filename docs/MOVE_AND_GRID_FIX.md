# 🔧 移动和网格修复指南

**版本**: v0.3.6-alpha  
**更新时间**: 2026-03-14

---

## ❌ 问题报告

### 问题 1: 地面看不到网格
**症状**: 地面是纯色，没有参照物，不知道角色是否移动

**原因**: 网格线太细太暗，看不清

---

### 问题 2: 角色无法移动
**症状**: 右键点击地面，角色没有反应

**可能原因**:
1. Canvas 没有 focus，无法接收键盘/鼠标事件
2. 玩家数据未初始化
3. 网络发送失败
4. 输入监听器未正确设置

---

## ✅ 修复方案

### 修复 1: 增强地面网格

**修改**: `client/src/systems/MapSystem.ts`

**新网格特征**:
- ✅ **白色边框** - 非常明显
- ✅ **十字线** - 辅助看清移动方向
- ✅ **中心红点** - 定位参考
- ✅ **四角白点** - 辅助定位

**效果**:
```
┌───────────────┐
│ ●           ● │
│               │
│       +       │
│               │
│ ●           ● │
└───────────────┘
```

---

### 修复 2: Canvas 自动 Focus

**修改**: `client/src/renderer/GameRenderer.ts`

**添加**:
```typescript
canvas.tabIndex = 0
canvas.style.outline = 'none'
setTimeout(() => canvas.focus(), 100)
```

**效果**: Canvas 自动获得焦点，可以接收键盘/鼠标事件

---

### 修复 3: 移动调试增强

**修改**: `client/src/systems/PlayerControlsSystem.ts`

**添加日志**:
- Canvas 元素信息
- Focus 状态
- 右键点击坐标
- 玩家位置
- 移动目标

---

### 修复 4: 诊断工具

**新增文件**: `client/public/diagnose-move.html`

**功能**:
- ✅ 系统状态检查
- ✅ Canvas 元素检测
- ✅ WebSocket 连接测试

## 🐛 故障排查

### 问题 A: 右键点击没日志

**检查**:
1. 按 F12 打开控制台
2. 切换到 "Console" 标签
3. 右键点击地面
4. 看是否有日志

**如果没日志**:
```javascript
// 在控制台输入以下代码手动测试
var canvas = document.querySelector('canvas');
console.log('Canvas:', canvas ? '存在' : '不存在');
console.log('Focus:', document.activeElement === canvas);
canvas.focus();
```

---

### 问题 B: 有日志但角色不动

**检查控制台**:
```
🚶 移动中：{ x: 410, y: 300, ... }  ← 有这个说明本地移动正常
📍 移动已发送：dx=10, dy=0  ← 有这个说明网络发送正常
```

**如果本地移动但服务器没响应**:
- 检查服务端日志
- 检查 WebSocket 连接状态

---

### 问题 C: 看不到玩家

**检查控制台**:
```
👤 玩家数据：{ id: '...', x: 400, y: 300 }
⚔️ 玩家精灵已创建
```

**如果没有玩家数据**:
- 刷新页面重新登录
- 检查登录是否成功
