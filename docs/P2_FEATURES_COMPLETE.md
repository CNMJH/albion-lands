## 🎯 详细功能说明

### 1. 成就系统

**API 端点**:
- `GET /api/v1/achievements/:characterId` - 获取角色成就
- `POST /api/v1/achievements/update` - 更新成就进度

**成就类型** (7 种):
1. **怪物猎手** - 累计击杀怪物 (10/50/100/500/1000)
2. **PVP 勇士** - 累计击杀玩家 (1/5/10/50/100)
3. **不屈意志** - 累计死亡次数 (10/50/100/500/1000)
4. **成长之路** - 角色等级 (10/20/30/50/100)
5. **财富积累** - 累计银币 (10k/50k/100k/500k/1000k)
6. **工匠大师** - 累计制造物品 (10/50/100/500/1000)
7. **采集专家** - 累计采集资源 (50/200/500/2000/5000)

**奖励机制**:
- 每阶成就发放银币 + 经验奖励
- 自动检测进度并发放
- 成就完成状态永久保存

---

### 2. 排行榜系统

**API 端点**:
- `GET /api/v1/leaderboard/level` - 等级排行榜
- `GET /api/v1/leaderboard/pvp` - PVP 排行榜
- `GET /api/v1/leaderboard/wealth` - 财富排行榜
- `GET /api/v1/leaderboard/kills` - 击杀排行榜
- `GET /api/v1/leaderboard/my-rank/:characterId` - 我的排名

**功能特性**:
- 前 50 名展示
- 实时排名计算
- 百分位统计
- KDA 计算（击杀 + 助攻）/ 死亡

---

### 3. 仓库系统（银行）

**API 端点**:
- `GET /api/v1/bank/:characterId` - 获取仓库物品
- `POST /api/v1/bank/deposit` - 存入物品
- `POST /api/v1/bank/withdraw` - 取出物品

**功能特性**:
- 100 格仓库容量
- 与背包物品互通
- 事务保证安全（原子操作）
- 堆叠物品支持
- 容量检测（背包/仓库）

**安全机制**:
- Prisma Transaction 保证原子性
- 双向验证（背包有货/仓库有货）
- 容量限制防止溢出

---

### 4. 断线重连系统

**API 端点**:
- `POST /api/v1/player/disconnect` - 玩家断线
- `POST /api/v1/player/reconnect` - 玩家重连
- `GET /api/v1/player/offline-rewards/:characterId` - 获取离线奖励
- `POST /api/v1/player/claim-offline-rewards` - 领取离线奖励

**功能特性**:
- 断线保存当前位置
- 重连恢复在线状态
- 离线奖励累计
- 24 小时上限保护

**离线奖励公式**:
```
每小时奖励 = 10 银币 + 50 经验
最大累计 = 24 小时（240 银币 + 1200 经验）
```

### 5. 物品详情系统

**API 端点**:
- `GET /api/v1/items/:itemId` - 获取物品详情
- `GET /api/v1/items/compare?itemId1=xxx&itemId2=yyy` - 物品比较

**功能特性**:
- 完整物品信息展示
- 市场价格统计（最低/最高/平均）
- 拥有者数量统计
- 市场订单列表
- 物品属性对比

**市场数据**:
- 最低价格
- 最高价格
- 平均价格
- 上架数量
- 拥有者数量

## 🔧 技术实现要点

### 1. 事务安全
所有涉及物品转移的操作都使用 Prisma Transaction：
```typescript
await prisma.$transaction(async (tx) => {
  // 背包减少
  await tx.inventoryItem.update(...)
  // 仓库增加
  await tx.bankItem.create(...)
})
```

### 2. 成就进度追踪
自动检测成就完成状态并发放奖励：
```typescript
for (let i = targets.length - 1; i >= 0; i--) {
  if (progress >= targets[i] && !completed) {
    // 发放奖励
    await prisma.character.update({
      data: { silver: { increment }, exp: { increment } }
    })
  }
}
```

### 3. 排行榜性能优化
使用 Prisma 的 orderBy + take 限制查询数量：
```typescript
const leaderboard = await prisma.character.findMany({
  orderBy: { level: 'desc' },
  take: 50  // 只查前 50 名
})
```

### 4. 离线奖励计算
基于时间差的动态计算：
```typescript
const offlineSeconds = Math.floor((Date.now() - lastLogin) / 1000)
const cappedSeconds = Math.min(offlineSeconds, 86400)  // 最多 24 小时
```
