# 呼噜大陆 (Hulu Lands) - 精简版游戏设计文档

**版本：** 2.0 (H5 精简版)  
**创建日期：** 2026-03-11  
**定位：** 简单易上手的类阿尔比恩 H5 页游  
**核心原则：** 保留核心精华，砍掉复杂系统，H5 优先适配

---

# 📋 设计哲学

## 一、阿尔比恩核心精华（必须保留）

```
┌─────────────────────────────────────────┐
│         阿尔比恩四大核心支柱             │
├─────────────────────────────────────────┤
│ ✅ 玩家驱动经济 - 采集→制造→交易闭环    │
│ ✅ 装备决定职业 - 无固定职业，自由搭配  │
│ ✅ 风险分级区域 - 高风险高回报          │
│ ✅ 死亡掉落机制 - PVP 有代价，刺激      │
└─────────────────────────────────────────┘
```

## 二、简化/砍掉的系统（性价比低）

| 系统 | 阿尔比恩 | 呼噜大陆 H5 | 理由 |
|------|----------|-------------|------|
| 公会领土战 | 50v50 | ❌ 砍掉 | H5 性能不支持 |
| 复杂副本 | 多层地下城 | ❌ 砍掉 | 开发成本高 |
| 多货币系统 | 金银铜 + 金币 | ✅ 简化为 2 种 | 降低理解成本 |
| 声望系统 | 多阵营声望 | ❌ 砍掉 | 初期不需要 |
| 复杂制造树 | 数百配方 | ✅ 简化为 50 种 | 快速上手 |
| 赛季系统 | 季度重置 | ❌ 后期考虑 | 初期不需要 |
| 跨平台 | PC+ 移动端 | ✅ H5 优先 | 专注单一平台 |

## 三、H5 页游适配原则

```
┌─────────────────────────────────────────┐
│           H5 页游设计原则                │
├─────────────────────────────────────────┤
│ 1. 加载快 - 首屏<5 秒，资源分包懒加载    │
│ 2. 操作简单 - 点击移动，一键操作        │
│ 3. 性能优 - 同屏<20 角色，简化特效      │
│ 4. 断线友好 - 自动重连，离线保护        │
│ 5. 碎片化 - 单局 5-15 分钟，随时可停    │
└─────────────────────────────────────────┘
```

  const delta = calculateDelta(newState, oldState);
  sendDelta(delta); // 只发送差异
}

// 视野剔除（只同步可见实体）
function getVisibleEntities(player, range = 20) {
  return entities.filter(e => distance(player, e) < range);
}
