### 2. UI 互斥原则违反 ❌

**设计文档要求** (PROFILE.md 约束):
> UI 互斥原则：同时只打开一个 UI 面板，打开新面板自动关闭旧面板

**实测结果**:
```
测试步骤:
1. 按 B 键 → 背包打开 ✅
2. 按 C 键 → 装备打开，但背包未关闭 ❌
3. 按 M 键 → 拍卖行打开，背包和装备仍未关闭 ❌

结果：3 个 UI 面板同时显示，违反互斥原则
```

**影响**: 
- 界面混乱，遮挡游戏画面
- 用户体验差
- 违反设计规范

**修复方案**:
```typescript
// PlayerControlsSystem.ts toggleUI() 方法
function toggleUI(panelType: UIType) {
  // 关闭所有其他 UI
  Object.keys(uiState).forEach(key => {
    if (key !== panelType) {
      uiState[key] = false
    }
  })
  
  // 切换目标 UI
  uiState[panelType] = !uiState[panelType]
  
  // 同步到 gameStore
  gameStore.setUIState(uiState)
}
```

**优先级**: 🔴 P0 (严重影响体验)

### 3. 智能攻击系统不完整 ⚠️

**设计文档要求** (GAME_DESIGN_LITE.md P892-950):

#### 3.1 自动攻击流程

```
设计需求:
1. 玩家右键点击敌人
2. 检测点击位置是否有敌人（50px 范围）
3. 计算到敌人的距离
4a. 在攻击范围内 → 直接攻击
4b. 超出攻击范围 → 移动到攻击范围边缘
5. 到达攻击位置后自动攻击
6. 持续追踪目标（如果目标移动）

当前状态:
1. ✅ 右键点击移动/攻击
2. ✅ 检测敌人
3. ✅ 计算距离
4. ✅ 移动逻辑
5. ⚠️ 自动攻击（需要手动再次点击）
6. ❌ 目标追踪（目标移动后丢失）
```

**影响**: 
- 需要多次点击才能攻击
- 怪物移动后需要重新锁定
- 操作繁琐，不符合 LOL 风格

**建议修复**:
- 实现自动攻击持续逻辑
- 添加目标追踪系统
- 优化攻击范围检测

**优先级**: 🟡 P1 (影响战斗体验)
