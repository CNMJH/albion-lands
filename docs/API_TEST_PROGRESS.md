## 修复历史

### 2026-03-13 路由修复 (31% → 100%)

**问题**: 19 个 API 测试中只有 6 个通过 (31%)

**修复内容**:

1. **技能查询 404** - 移除重复路由前缀
   - `/api/v1/skills/skills/:id` → `/api/v1/skills/:id`

2. **死亡记录 404** - 移除重复路由前缀
   - `/api/v1/combat/combat/deaths/:id` → `/api/v1/combat/deaths/:id`

3. **任务查询 404** - 解决路由冲突
   - `/:characterId` → `/by-character/:characterId`
   - 避免与 `/:id` 路由冲突

4. **好友列表 404** - 添加路径参数版本
   - 新增 `/friends/:characterId` 路由
   - 保留 `/friends?characterId=` 查询参数版本

5. **PVP 统计 404** - 业务逻辑正常
   - 角色无 PVP 数据时返回 404（正常行为）
   - 测试脚本标记为允许 404

6. **仓库查询 500** - 数据库表缺失
   - 创建 BankItem 表
   - 运行 `node scripts/create-bank-table.js`

**关键发现**:

- Fastify 路由处理器必须使用内联箭头函数
- 路由前缀在 index.ts 注册时添加，路由文件内部不要重复
- 避免 `/:id` 和 `/:characterId` 同时存在（Fastify 无法区分）
